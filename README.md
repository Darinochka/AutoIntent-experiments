- ~~достает все метрики качества для всех скореров (каждый скорер представлен своей лучшей конфигурацией гиперпараметров) и строит таблицу~~ (и мб графики)
- достает все метрики производительности (carbon code, время, интегральные метрики CPU, GPU, RAM) и строит таблицу (и мб графики)

В этой ветке находится код для сравнения качества ho и cv валидаций. Для каждого метода мы будем считать финальные метрики для нескольких сидов и смотреть насколько оно зависит от схемы выбора гиперпараметров.

Можно провести еще вот такие эксперименты, похожие на описанный выше:
- качество разных семплеров (brute, random, tpe)
- использовать разделение данных между двумя слоями (`0 < separation_ratio < 1` для scoring и decision) или не использовать (`separation_ratio=None`)
- прогнать все пресеты

То есть это все эксперименты которые оценивают качество различных стратегий поиска гиперпараметров.

Дополнительно из каждого эксперимента можно посмотреть качество каждого скорера. Интересно также посмотреть, отдают ли разные стратегии поиска гиперпараметров конкретным скорерам.

Еще эксперимент который можно вычленить из этих экспериментов:
- какой большой разброс в качестве различных алгоритмов от конфигурации к конфигурации (это будет характеризовать чувствительность каждого метода)
- отличаются ли оптимальные конфигурации от датасета к датасету (или можно выделить какой-то эталонный набор гиперпараметров который хорош для любого датасета; если можно то было бы классно включить его в библиотеку)
